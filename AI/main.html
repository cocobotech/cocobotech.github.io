Transformer 架构

2017年，Google 团队发表了论文 "Attention Is All You Need"，彻底改变了自然语言处理（NLP）领域。Transformer 摒弃了传统的循环神经网络（RNN），完全基于注意力机制。


  • 并行计算： 可以同时处理整个句子，而不是逐字处理。
  • 长距离依赖： 无论单词相距多远，都能直接建立联系。

自注意力机制 (Self-Attention)

核心公式

注意力机制的核心是计算 Query (Q), Key (K), Value (V) 之间的关系。

$$ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

可以将这理解为一个数据库查询：
1. Query (Q): 你在寻找什么。
2. Key (K): 数据库中的标签。
3. Score ($QK^T$): Q 和 K 的匹配度（相似度）。
4. Value (V): 实际提取的内容。

多头注意力 (Multi-Head Attention)

为了捕捉不同维度的语义信息（如：语法关系、指代关系、语气等），Transformer 将 Q, K, V 拆分成多个“头”并行处理。

解释： 输入数据被“分流”进多个 Head，每个 Head 关注输入序列的不同特征（比如 Head 1 关注动词，Head 2 关注代词），最后将结果拼接，得到完整理解。

从 BERT 到 GPT-4

  • Encoder-Only (BERT)
    擅长“理解”任务，如情感分析、文本分类。只使用了 Transformer 的编码器部分。
  • Decoder-Only (GPT)
    擅长“生成”任务。通过预测下一个 token 来生成文本，构建了当今的 ChatGPT 等模型。
